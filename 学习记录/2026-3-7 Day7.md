# Day7：测试管理工具

---

# 一、为什么需要测试管理工具

在真实项目中：

一个系统可能有：

几百个测试用例  
几十个 Bug

如果只用 Excel 或文档：

会非常混乱。

因此公司会使用测试管理工具。

---

# 二、常见测试管理工具

常见工具：

*Jira*  
*禅道*  
*TestRail*

---

## Jira 特点

国际公司常用

功能：

- 需求管理  
- Bug管理  
- 任务管理  
- 测试管理

---

## 禅道特点

国内很多公司使用

优点：  
开源  
功能完整  
适合中小团队

---

# 三、测试人员在工具中的工作

测试人员通常需要：

- 创建测试用例  
- 执行测试用例  
- 提交 Bug  
- 跟踪 Bug 状态

---

# 四、Bug 在工具中的流程

典型流程：

New  
Assigned  
Open  
Fixed  
Retest  
Closed

---

# 五、实际工作流程

真实公司流程：

产品经理 → 提需求  
开发 → 写代码  
测试 → 写测试用例  
测试 → 执行测试  
测试 → 提 Bug  
开发 → 修 Bug  
测试 → 回归测试

---

# 今日重点总结

测试管理工具用于：

管理测试用例  
管理 Bug  
跟踪项目进度

测试工程师每天都会使用这些工具。

# Day7 作业

## 问题1

为什么公司需要测试管理工具？

你的答案：因为在真实项目中,一个系统可能有：几百个测试用例、几十个Bug如果只用Excel或文档会非常混乱。

    因为在真实项目中，一个系统可能有大量测试用例和Bug，如果只使用Excel或文档管理会非常混乱。
    测试管理工具可以帮助团队统一管理测试用例、Bug以及测试进度，提高团队协作效率。

## 问题2

测试工程师在测试管理工具中通常做哪些事情？

你的答案：管理测试用例  、管理 Bug  、跟踪项目进度

    测试工程师在测试管理工具中通常会管理测试用例、提交和跟踪Bug、记录测试执行结果以及跟踪项目测试进度。
