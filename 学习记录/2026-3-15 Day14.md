# Day14：完整接口测试流程

---

# 一、真实公司的接口测试流程

企业做接口测试一般是：

```
需求分析
↓
接口文档分析
↓
设计测试用例
↓
Postman测试接口
↓
编写断言
↓
提交Bug
```

---

# 二、需求分析

第一步必须看：

```
需求文档
```

例如需求：

```
用户可以登录系统
```

输入：

```
username
password
```

输出：

```
登录成功
返回 token
```

---

# 三、分析接口文档

接口文档示例：

```
POST /login
```

参数：

| 参数       | 类型     | 必填 |
|----------|--------|----|
| username | string | 是  |
| password | string | 是  |

返回：

```
{
 "code":200,
 "msg":"login success",
 "token":"abc123"
}
```

---

# 四、设计接口测试用例

例如：

| 用例ID | 测试场景  | 输入           | 预期     |
|------|-------|--------------|--------|
| TC01 | 正常登录  | admin/123456 | 登录成功   |
| TC02 | 密码错误  | admin/111111 | 登录失败   |
| TC03 | 用户名为空 | ""/123456    | 提示不能为空 |

---

# 五、Postman 发送接口请求

例如：

```
POST /login
```

Body：

```
{
 "username":"admin",
 "password":"123456"
}
```

点击：

```
Send
```

查看返回数据。

---

# 六、添加接口断言

在 Tests 中写：

```javascript
pm.response.to.have.status(200);

var jsonData = pm.response.json();

pm.expect(jsonData.code).to.eql(200);
pm.expect(jsonData.msg).to.eql("login success");
pm.expect(jsonData.token).to.exist;
```

---

# 七、发现 Bug

例如：

需求：

```
密码必须6位以上
```

测试：

```
password = 123
```

结果：

```
登录成功
```

说明：

```
存在Bug
```

需要提交：

```
Bug报告
```

# 九、今日练习题

假设有一个接口：

```
POST /order/create
```

参数：

```
userId
productId
count
```

要求：

```
userId 必须存在
productId 必须存在
count > 0
```

请设计 **5条接口测试用例**

---

答：

| 用例ID | 测试场景        | 输入               | 预期              |
| ---- | ----------- | ---------------- | --------------- |
| TC01 | 正常创建订单      | admin/111/2      | 创建成功            |
| TC02 | userId为空    | ""/111/2         | 提示userId不能为空    |
| TC03 | productId为空 | admin/""/2       | 提示productId不能为空 |
| TC04 | count<0     | admin/111/-1     | 提示count必须>0     |
| TC05 | count=0     | admin/111/0      | 提示count必须>0     |
| TC06 | count类型错误   | admin/111/"abc"  | 提示参数错误          |
| TC07 | count超大     | admin/111/999999 | 系统正常处理          |
