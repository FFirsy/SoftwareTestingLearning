# 一、什么是 HTTP

HTTP（HyperText Transfer Protocol）  
中文：**超文本传输协议**

它是一种：

- 客户端 ↔ 服务器 的通信协议
- 用于 **浏览器与服务器之间的数据传输**

例如：

你在浏览器输入：

```
https://www.baidu.com
```

浏览器就会发送 **HTTP请求** 给服务器。

服务器再返回 **HTTP响应**。

流程：

```
客户端（浏览器/Postman/代码）
        ↓ HTTP请求
服务器（Web Server）
        ↑ HTTP响应
客户端接收数据
```

---

# 二、HTTP 请求结构

一个 HTTP 请求主要由 **4部分组成**

```
请求行
请求头
空行
请求体
```

示例：

```
POST /login HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "username":"admin",
  "password":"123456"
}
```

解释：

## 1. 请求行

```
POST /login HTTP/1.1
```

包含：

|部分|说明|
|---|---|
|POST|请求方法|
|/login|请求路径|
|HTTP/1.1|协议版本|

---

## 2. 请求头（Header）

例如：

```
Host: example.com
Content-Type: application/json
User-Agent: Chrome
```

作用：

告诉服务器一些信息，例如：

- 发送者是谁
- 数据格式是什么
- 浏览器类型

常见 Header：

|Header|作用|
|---|---|
|Host|服务器地址|
|Content-Type|数据类型|
|User-Agent|客户端类型|
|Authorization|认证信息|

---

## 3. 空行

Header 和 Body 的分隔。

---

## 4. 请求体（Body）

例如：

```
{
 "username":"admin",
 "password":"123456"
}
```

一般用于：

- POST
- PUT
- PATCH

发送数据。

---

# 三、HTTP 响应结构

服务器返回的数据结构：

```
状态行
响应头
空行
响应体
```

示例：

```
HTTP/1.1 200 OK
Content-Type: application/json

{
 "msg":"login success"
}
```

---

## 1. 状态行

```
HTTP/1.1 200 OK
```

包含：

|部分|说明|
|---|---|
|HTTP/1.1|协议版本|
|200|状态码|
|OK|状态描述|

---

## 2. 响应头

例如：

```
Content-Type: application/json
Set-Cookie: sessionid=123456
```

---

## 3. 响应体

服务器真正返回的数据。

例如：

```
HTML
JSON
图片
文件
```

---

# 四、常见 HTTP 请求方法

测试岗位 **最重要的是这4个**

|方法|作用|
|---|---|
|GET|获取数据|
|POST|创建数据|
|PUT|更新数据|
|DELETE|删除数据|

---

## 1 GET

用于：

获取资源

例如：

```
GET /users
```

特点：

- 参数在 URL
- 没有 body

示例：

```
/users?id=1
```

---

## 2 POST

用于：

创建资源

例如：

```
POST /login
```

数据一般在：

```
Body
```

示例：

```
{
 "username":"admin",
 "password":"123456"
}
```

---

## 3 PUT

用于：

更新资源

例如：

```
PUT /user/1
```

---

## 4 DELETE

用于：

删除资源

例如：

```
DELETE /user/1
```

---

# 五、GET 和 POST 的区别（面试高频）

|区别|GET|POST|
|---|---|---|
|用途|获取数据|提交数据|
|参数位置|URL|Body|
|安全性|低|稍高|
|长度限制|有|基本没有|
|缓存|会缓存|不会缓存|

面试经典问题：

```
GET 和 POST 有什么区别？
```

# 六、为什么测试必须懂 HTTP

接口测试本质就是：

```
构造 HTTP 请求
验证 HTTP 响应
```

例如：

登录接口：

```
POST /login
```

发送：

```
{
 "username":"admin",
 "password":"123456"
}
```

验证：

```
status code = 200
msg = login success
```
