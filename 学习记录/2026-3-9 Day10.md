# Day10 — Postman 接口测试基础

---

# 一、什么是 Postman

Postman 是：

```
最常用的接口测试工具
```

主要功能：

- 发送 HTTP 请求
- 查看接口返回数据
- 调试接口
- 进行接口测试

测试工程师常用它来：

```
测试 API 接口
```

---

# 二、Postman 能做什么

Postman 可以：

| 功能    | 说明                        |
|-------|---------------------------|
| 发送请求  | GET / POST / PUT / DELETE |
| 调试接口  | 查看请求和响应                   |
| 测试接口  | 验证接口返回数据                  |
| 自动化测试 | 写测试脚本                     |

---

# 三、Postman 界面介绍

主要分为 4 个区域：

```
请求方法
请求地址
请求参数
响应结果
```

示意：

```
[请求方法]  GET / POST
[请求地址]  https://api.test.com/login

Body / Params / Headers

Response
```

---

# 四、发送 GET 请求

GET 请求一般用于：

```
获取数据
```

示例接口：

```
GET https://jsonplaceholder.typicode.com/users
```

步骤：

1 打开 Postman

2 选择请求方法

```
GET
```

3 输入 URL

```
https://jsonplaceholder.typicode.com/users
```

4 点击：

```
Send
```

返回示例：

```
[
 {
  "id":1,
  "name":"Leanne Graham"
 },
 {
  "id":2,
  "name":"Ervin Howell"
 }
]
```

---

# 五、发送 POST 请求

POST 请求用于：

```
提交数据
```

示例接口：

```
POST https://jsonplaceholder.typicode.com/posts
```

步骤：

1 选择：

```
POST
```

2 输入 URL

```
https://jsonplaceholder.typicode.com/posts
```

3 点击：

```
Body
```

4 选择：

```
raw
```

5 选择数据格式：

```
JSON
```

6 输入 JSON 数据：

```
{
 "title":"test",
 "body":"hello",
 "userId":1
}
```

7 点击：

```
Send
```

返回示例：

```
{
 "id":101,
 "title":"test",
 "body":"hello",
 "userId":1
}
```

---

# 六、Postman 常用参数位置

接口参数常见有三种：

| 位置      | 说明    |
|---------|-------|
| Params  | URL参数 |
| Headers | 请求头   |
| Body    | 请求体   |

---

## 1 Params（URL参数）

示例：

```
GET /users?id=1
```

在 Postman：

```
Params
key: id
value: 1
```

---

## 2 Headers（请求头）

示例：

```
Content-Type: application/json
Authorization: token
```

作用：

```
告诉服务器请求信息
```

---

## 3 Body（请求体）

一般用于：

```
POST
PUT
PATCH
```

示例：

```
{
 "username":"admin",
 "password":"123456"
}
```

---

# 七、接口测试基本验证

发送请求后需要验证：

| 检查项  | 示例             |
|------|----------------|
| 状态码  | 200            |
| 返回字段 | code / msg     |
| 字段值  | msg == success |
| 数据结构 | JSON格式         |

例如返回：

```
{
 "code":200,
 "msg":"success"
}
```

验证：

```
status_code == 200
msg == success
```
