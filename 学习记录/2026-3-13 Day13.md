# Day13：接口测试用例设计

---

# 一、什么是接口测试用例

接口测试用例：

```
用于验证接口是否正确工作的测试场景
```

例如接口：

```
POST /login
```

参数：

```
username
password
```

返回：

```
{
 "code":200,
 "msg":"login success"
}
```

测试需要设计：

```
不同输入场景
```

---

# 二、接口测试主要测试什么

接口测试需要验证：

| 检查项  | 说明       |
|------|----------|
| 功能正确 | 接口是否正常工作 |
| 参数验证 | 参数是否合法   |
| 异常处理 | 异常情况是否正确 |
| 安全性  | 是否存在漏洞   |

---

# 三、接口测试常见场景

测试接口通常要设计以下几类用例：

| 类型     | 示例           |
|--------|--------------|
| 正常用例   | 正确参数         |
| 错误参数   | 密码错误         |
| 参数缺失   | 不传 password  |
| 参数类型错误 | password=123 |
| 边界值    | 用户名长度边界      |
| 安全测试   | SQL注入        |

---

# 四、登录接口测试示例

接口：

```
POST /login
```

参数：

```
username
password
```

---

## 1 正常登录

请求：

```
username = admin
password = 123456
```

预期：

```
登录成功
返回 token
```

---

## 2 密码错误

请求：

```
username = admin
password = 111111
```

预期：

```
登录失败
提示密码错误
```

---

## 3 用户名错误

请求：

```
username = test
password = 123456
```

预期：

```
用户不存在
```

---

## 4 参数缺失

请求：

```
username = admin
password = ""
```

预期：

```
提示参数不能为空
```

---

## 5 参数类型错误

请求：

```
username = 123
password = true
```

预期：

```
参数类型错误
```

---

## 6 SQL注入测试

请求：

```
username = ' OR 1=1 --
password = 123456
```

预期：

```
登录失败
```

# 七、今日练习题

假设有一个注册接口：

```
POST /register
```

参数：

```
username
password
email
```

要求：

```
username 不为空
password 6-12位
email 必须合法
```

请设计 **5条接口测试用例**。
