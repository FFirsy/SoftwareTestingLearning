# Day9 — JSON 与 API接口基础

---

# 一、什么是 JSON

JSON（JavaScript Object Notation）

中文：**JavaScript 对象表示法**

它是一种：

```
轻量级的数据交换格式
```

特点：

- 易读
- 易解析
- 跨语言
- 接口最常见的数据格式

在 **接口测试中**

服务器通常返回：

```
JSON数据
```

例如：

```
{
 "code":200,
 "msg":"success",
 "data":{
     "username":"admin",
     "age":18
 }
}
```

---

# 二、JSON 的基本结构

JSON 有两种结构：

```
对象（Object）
数组（Array）
```

---

## 1 JSON 对象

对象使用：

```
{}
```

结构：

```
{
 "键":值
}
```

示例：

```
{
 "username":"admin",
 "age":18
}
```

说明：

|元素|说明|
|---|---|
|username|键（key）|
|admin|值（value）|

注意：

```
key 必须是字符串
```

---

## 2 JSON 数组

数组使用：

```
[]
```

示例：

```
[
 "apple",
 "banana",
 "orange"
]
```

或者对象数组：

```
[
 {
  "id":1,
  "name":"张三"
 },
 {
  "id":2,
  "name":"李四"
 }
]
```

---

# 三、JSON 常见数据类型

| 类型  |示例|
|-----|---|
| 字符串 |string|
| 数字  |18|
| 布尔  |true / false|
| 数组  |[]|
| 对象  |{}|
| 空   |null|

示例：

```
{
 "name":"admin",
 "age":18,
 "is_login":true,
 "hobby":["game","music"],
 "info":{
   "city":"beijing"
 },
 "data":null
}
```

---

# 四、什么是 API 接口

API：

```
Application Programming Interface
```

中文：

```
应用程序接口
```

简单理解：

```
系统提供给外部调用的功能入口
```

例如：

| 功能     |接口|
|--------|---|
| 用户登录   |POST /login|
| 获取用户信息 |GET /user|
|  创建订单  |POST /order|

---

# 五、接口调用流程

接口调用过程：

```
客户端
  ↓
发送 HTTP 请求
  ↓
服务器
  ↓
处理业务逻辑
  ↓
返回 JSON 数据
  ↓
客户端解析数据
```

示例：

登录接口

请求：

```
POST /login
```

Body：

```
{
 "username":"admin",
 "password":"123456"
}
```

返回：

```
{
 "code":200,
 "msg":"login success",
 "token":"abc123"
}
```

---

# 六、接口测试主要做什么

接口测试核心：

```
验证接口返回数据是否正确
```

主要检查：

| 检查项   |说明|
|-------|---|
| 状态码   |是否200|
| 返回字段  |是否存在|
| 字段值   |是否正确|
| 业务逻辑  |是否符合预期|
| 异常情况 |是否正确处理|

例如：

测试登录接口：

```
POST /login
```

发送：

```
{
 "username":"admin",
 "password":"123456"
}
```

验证：

```
status_code == 200
msg == "login success"
token 不为空
```

---

# 七、接口字段解释（非常重要）

例如接口返回：

```
{
 "code":200,
 "msg":"success",
 "data":{
    "id":1,
    "username":"admin"
 }
}
```

字段含义：

| 字段   | 含义    |
|------|-------|
| code | 业务状态码 |
| msg  | 提示信息  |
| data | 业务数据  |

测试需要验证：

```
code == 200
msg == success
data 不为空
```
