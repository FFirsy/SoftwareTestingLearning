# Day11 — Postman 接口断言（Tests）

---

# 一、什么是接口断言

接口断言：

```
验证接口返回结果是否符合预期
```

简单理解：

```
自动检查接口返回数据
```

例如接口返回：

```
{
 "code":200,
 "msg":"success"
}
```

测试需要验证：

```
code == 200
msg == success
```

这就叫：

```
断言（Assertion）
```

---

# 二、Postman 中的 Tests

Postman 有一个功能：

```
Tests
```

作用：

```
编写测试脚本验证接口结果
```

位置：

```
Postman → Tests
```

执行顺序：

```
发送请求
↓
接口返回数据
↓
执行 Tests 脚本
↓
显示测试结果
```

---

# 三、验证状态码（最常见）

示例代码：

```javascript
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});
```

作用：

```
验证 HTTP 状态码是否为 200
```

如果不是 200：

```
测试失败
```

---

# 四、获取接口返回 JSON

接口返回：

```
{
 "code":200,
 "msg":"success"
}
```

Postman 需要先获取 JSON：

```javascript
var jsonData = pm.response.json();
```

含义：

```
把返回数据解析为 JSON 对象
```

---

# 五、验证字段值

例如接口返回：

```
{
 "code":200,
 "msg":"success"
}
```

验证 code：

```javascript
pm.expect(jsonData.code).to.eql(200);
```

验证 msg：

```javascript
pm.expect(jsonData.msg).to.eql("success");
```

---

# 六、验证字段存在

有些时候需要检查字段是否存在。

例如：

```
token
```

代码：

```javascript
pm.expect(jsonData.token).to.exist;
```

意思：

```
token 字段必须存在
```

---

# 七、完整断言示例

接口返回：

```
{
 "code":200,
 "msg":"login success",
 "token":"abc123"
}
```

Tests 示例：

```javascript
pm.test("status code is 200", function () {
    pm.response.to.have.status(200);
});

var jsonData = pm.response.json();

pm.test("code is 200", function () {
    pm.expect(jsonData.code).to.eql(200);
});

pm.test("msg correct", function () {
    pm.expect(jsonData.msg).to.eql("login success");
});

pm.test("token exists", function () {
    pm.expect(jsonData.token).to.exist;
});
```

执行结果：

```
✓ status code is 200
✓ code is 200
✓ msg correct
✓ token exists
```

# 八、接口测试常见断言

测试工程师最常写的断言：

|断言类型|示例|
|---|---|
状态码|status == 200|
字段值|msg == success|
字段存在|token 存在|
字段类型|id 是数字|
返回时间|response time < 500ms|
