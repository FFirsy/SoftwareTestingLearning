# Day12：Postman 变量（Variables）

---

# 一、什么是变量

变量：

```
用于存储数据
```

例如：

```
token
userId
username
```

作用：

```
让接口之间共享数据
```

---

# 二、为什么需要变量

真实接口流程：

```
登录接口
POST /login
```

返回：

```
{
 "token":"abc123"
}
```

下一个接口：

```
GET /user/info
```

需要：

```
Authorization: abc123
```

如果不用变量：

```
每次手动复制 token
```

如果用变量：

```
自动获取 token
```

---

# 三、Postman 变量类型

Postman 常见变量：

|变量类型|说明|
|---|---|
Global|全局变量|
Collection|集合变量|
Environment|环境变量|

测试最常用：

```
Environment
```

---

# 四、设置变量

例如创建变量：

```
token
```

在 Postman：

```
Environment → Add
```

添加：

```
token = ""
```

---

# 五、获取接口返回数据并保存变量

例如登录接口返回：

```
{
 "token":"abc123"
}
```

在 Tests 中写：

```javascript
var jsonData = pm.response.json();

pm.environment.set("token", jsonData.token);
```

作用：

```
把 token 保存为环境变量
```

---

# 六、使用变量

使用方式：

```
{{变量名}}
```

例如：

```
{{token}}
```

Header 示例：

```
Authorization: {{token}}
```

这样：

```
自动带 token
```

---

# 七、完整流程示例

登录接口：

```
POST /login
```

返回：

```
{
 "token":"abc123"
}
```

Tests：

```javascript
var jsonData = pm.response.json();

pm.environment.set("token", jsonData.token);
```

用户接口：

```
GET /user/info
```

Header：

```
Authorization: {{token}}
```

这样：

```
自动使用登录 token
```
