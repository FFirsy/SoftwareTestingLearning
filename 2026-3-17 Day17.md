# Pytest Framework

Pytest is a popular testing framework for Python that makes it easy to write simple as well as scalable test cases. It supports fixtures, parameterization, and a variety of plugins that can enhance its functionality.

## Key Features of Pytest

- **Easy to Use**: Pytest is designed to be straightforward. Writing tests is simple, allowing for easy integration into any project.
- **Rich Plugin Architecture**: Pytest has a large number of plugins available for extending its capabilities, from reports to integration with CI/CD pipelines.
- **Fixtures**: Pytest allows you to define setup code via fixtures that enhance your tests by providing a way to prepare prerequisites before the tests run.
- **Parameterized Testing**: This feature enables you to run the same test function with different input parameters, making it easy to test edge cases and a variety of inputs.

## Getting Started with Pytest

1. **Installation**: You can install pytest via pip:
   
   ```bash
   pip install pytest
   ```
2. **Writing a Test**: A simple test can be written in a file named `test_sample.py`:
   
   ```python
   def test_addition():
       assert 1 + 1 == 2
   ```
3. **Running Tests**: You can run the tests in the terminal using the command:
   
   ```bash
   pytest
   ```

## API Automation Testing with Pytest

API testing is crucial for ensuring that an application can communicate with other applications and services. Pytest can also be utilized for API automation testing. Here’s how to leverage it for API testing:

1. **Tools You Might Need**:
   - `requests` for making API calls
   - `pytest` for structuring and running tests

2. **Basic Example**:
   
   ```python
   import requests
   import pytest

   @pytest.mark.parametrize('endpoint, expected_status', [
       ('https://api.example.com/resource1', 200),
       ('https://api.example.com/resource2', 404),
   ])
   def test_api_calls(endpoint, expected_status):
       response = requests.get(endpoint)
       assert response.status_code == expected_status
   ```

3. **Running the Tests**: Just like any other python file, run `pytest` in your terminal to execute the API tests.

## Conclusion

Pytest provides an easy way to conduct both unit testing and API testing, ensuring that your code runs as intended and interacts appropriately with other services.
